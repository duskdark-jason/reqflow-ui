# 项目接入中心职责收敛 需求说明

## 背景

项目维护页已经承担项目基础信息、代码仓库、项目分支、初始化指令复制和初始化摘要展示能力。当前项目列表仍提供“接入”入口进入项目接入中心，而接入中心继续展示“复制初始化指令”等与维护页重复的动作，导致用户难以判断两个页面的职责差异。

本需求按已确认的方案一处理：保留项目接入中心，但移除或弱化重复的配置型能力，让项目维护页负责“配置与指令”，让项目接入中心负责“状态、验收和观察”。

## 目标

- 项目列表入口语义从“接入动作”调整为“接入状态/接入中心”，避免误导用户以为点击按钮会自动完成接入。
- 项目维护页保留初始化指令复制主入口；项目接入中心不再把复制初始化指令作为主操作。
- 项目接入中心聚焦展示仓库索引状态、分支初始化状态、模块知识库、最近 commit、失败或待处理信息。
- 分支知识库详情继续作为深层页签，承担按 `projectId + variantId` 查看知识库和索引详情的能力。
- 前端 harness 文档同步更新页面职责边界，避免后续继续把维护页和接入中心做成重复入口。

## 范围

本次包含：

- 调整项目列表操作区的“接入”按钮命名和目标语义，建议改为“接入状态”或“接入中心”。
- 调整 `src/views/requirement/project/detail.vue` 的首屏结构和文案，把页面定位为只读状态面板。
- 将接入中心中的初始化指令复制从主路径降级为辅助动作，或仅在状态异常/未初始化时提供明确的“去维护页复制指令”入口。
- 强化接入中心对索引批次、模块知识库、分支完成度和待处理项的展示。
- 保留“知识库”页签入口，并确保它按项目分支过滤。
- 同步更新 `docs/ai-harness/modules/requirement-platform.md`、`docs/ai-harness/contracts/requirement-platform-ui.md` 和 `docs/domains/requirement-platform/README.md` 中关于页面职责的描述。

本次不包含：

- 不删除项目接入中心路由。
- 不把项目维护功能迁回接入中心。
- 不新增后端接口、数据库表或 SQL 脚本。
- 不改变 `initInstruction`、`actionToken`、`publish_repository_index` 的后端语义。
- 计划阶段不创建任务分支、不改业务代码、不提交、不进入 Review；用户后续授权执行后，按任务分支模式进入实现、Review 和完成态收口。

## 影响范围

- 接口/API：否，计划以现有 `/requirement/project/init/{projectId}`、`/requirement/index/batch/list` 和 `/requirement/index/module/tree` 展示数据为主；执行阶段如发现缺少失败原因或 harness 回写字段，再补后端 companion。
- 数据库/SQL：否。
- 权限/菜单：是，项目列表操作文案和接入中心页面语义会调整，权限仍沿用 `req:project:query` 和 `req:index:list`。
- 页面/交互：是，影响项目管理列表、项目接入中心和分支知识库入口。
- 导出/异步/任务：否。

## 契约与数据口径

- 项目列表一行对应一个项目，接入状态来自 `/requirement/project/init/{projectId}` 的 `initChecklist` 和项目分支行级统计。
- 项目维护页是配置型页面，负责项目、仓库、分支和初始化指令复制。
- 项目接入中心是只读状态型页面，读取项目初始化上下文、索引批次和模块知识库；新增、编辑和指令主复制回到项目维护页。
- 分支初始化完成口径沿用当前前端兜底：`totalModules > 0`、`indexedRepositoryCount > 0` 且 `unindexedRepositoryCount = 0`。
- 模块知识库必须按 `projectId + variantId` 过滤；旧 `variantId` 为空的数据不得默认混入选中分支。

## 验收标准

- AC-UI-001：项目列表操作按钮不再只叫“接入”，文案能明确这是状态或中心入口。
- AC-UI-002：项目接入中心首屏能直接看出项目、仓库、分支的接入完成度和待处理项，而不是以复制指令为主。
- AC-UI-003：初始化指令复制主入口保留在项目维护页；接入中心中的指令相关入口降级为辅助说明或跳转维护页。
- AC-UI-004：接入中心仍可查看索引批次、模块知识库和分支知识库详情，并按选中项目分支过滤。
- AC-UI-005：没有索引数据或模块知识时，接入中心展示明确的未完成状态和下一步建议，不把空表误判为完成。
- AC-UI-006：前端 harness 模块文档、契约文档和领域说明同步记录“维护=配置、接入中心=状态观察”的职责边界。
- AC-UI-007：构建和文档检查通过；执行阶段如具备后端环境，需要完成项目列表、维护页、接入中心和知识库详情的运行态冒烟。

## Companion 关联

- companion spec：无，当前计划不改后端接口。
- 关联分支：未创建，执行阶段建议使用 `feature/REQ-20260611-001-access-center-status`

## 客户与分支

- 目标客户：通用
- 基线分支：main
- 任务分支：feature/REQ-20260611-001-access-center-status

## 约束与假设

- 当前需求属于平台自身建设模式。
- 用户已选择“方案一”：保留项目接入中心，但砍掉重复编辑/指令主入口。
- 如果执行阶段发现现有接口无法展示 harness 初始化结果、失败原因或回写状态，应先补充计划并创建后端 companion spec，不得在前端伪造完成状态。
- 接入中心可以保留复制指令作为异常兜底，但不得继续作为页面主要价值点。
