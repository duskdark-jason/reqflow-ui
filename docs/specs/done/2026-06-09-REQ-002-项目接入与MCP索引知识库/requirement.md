# 项目接入与 MCP 索引知识库前端需求说明

## 背景

现有需求管理前端将项目、仓库、客户定制线、模块功能点、需求列表和需求执行包拆成多个独立菜单。这个形态能支撑 MVP-lite 的基础 CRUD，但不符合目标产品的项目接入体验：管理员希望在项目管理内完成项目创建、前后端仓库绑定、客户或地区基线维护，并通过本地索引器把模块功能点、页面、接口、数据表和权限沉淀到平台。需求人员创建新需求时，应通过选择功能模块自动获得影响面推荐，而不是手工填写所有影响页面和接口。

平台不应保存个人本机绝对路径。前端只在发起本地索引或展示索引指令时接收本次输入的本机目录，上传到平台或通过 MCP 推送的数据只包含 Git 远端、仓库类型、分支、commit、相对路径和结构化索引结果。

## 目标

- 将“项目管理”升级为项目接入中心，集中展示项目、仓库、客户基线、索引状态和模块知识库。
- 新增索引入口，引导管理员或开发者使用本地 Codex/索引 Agent 通过 MCP 推送索引结果。
- 展示索引批次、模块知识树和影响面条目，支持管理员校正。
- 创建或编辑需求时，选择项目、客户线和模块后自动加载推荐影响面。
- 将“需求执行包”在前端语义上调整为需求详情内的 Agent 交接资料，减少需求人员理解成本。

## 范围

本次包含：

- 调整需求管理菜单和页面入口，以项目接入中心承载项目、仓库、客户基线、索引状态和模块知识库。
- 新增或改造项目详情页，用 tab 展示基础信息、仓库、客户基线、索引批次、模块知识库。
- 新增索引指引 UI，说明 MCP 推送方式，并展示 `publish_repository_index` 所需项目 ID、仓库 ID 和注意事项。
- 新增影响面推荐 UI，在需求表单选择模块后自动填充或候选追加影响页面、接口、数据、权限和相关文档。
- 将执行包页面入口调整为需求详情内“Agent 交接资料”，页面仍复用现有 artifact 版本能力。
- 更新前端 harness 模块和接口调用契约。

本次不包含：

- 浏览器直接读取本机文件系统并扫描代码。
- 前端保存个人绝对路径到后端数据库。
- 前端实现完整代码解析器。
- 新增 Markdown 编辑器或复杂可视化依赖。
- 改变需求状态机和执行包版本追加规则。

## 影响范围

- 接口/API：是，新增索引批次、模块知识树、影响面推荐和可选 JSON 导入 API 调用。
- 数据库/SQL：否，前端不直接修改数据库；依赖后端 companion。
- 权限/菜单：是，项目接入中心、索引查询、索引导入和 Agent 交接资料入口需要权限联动。
- 页面/交互：是，调整项目管理、需求表单、需求详情和执行资料入口。
- 导出/异步/任务：否，本次不引入前端后台任务或文件导出。

## 契约与数据口径

- API 文件建议：
  - `src/api/requirement/index.js` 封装索引批次、模块知识树、影响面推荐和 JSON 导入备用接口。
  - 现有 `project.js`、`repository.js`、`variant.js`、`module.js` 继续保留基础 CRUD 调用。
- 页面入口建议：
  - `src/views/requirement/project/index.vue` 保留项目列表，但新增“进入接入中心”操作。
  - `src/views/requirement/project/detail.vue` 新增项目接入中心详情页。
  - `src/views/requirement/demand/index.vue` 在需求表单中接入影响面推荐。
  - `src/views/requirement/demand/detail.vue` 提供“Agent 交接资料”入口。
- 影响面推荐响应字段建议：
  - `pages`：页面名称、相对路径、路由或菜单。
  - `apis`：接口方法、路径、Controller 或服务。
  - `tables`：表名、字段、数据口径说明。
  - `permissions`：菜单权限、按钮权限或服务端权限标识。
  - `documents`：相关 harness 文档相对路径和摘要。
- 数据粒度：
  - 索引批次列表一行代表一次仓库索引上传。
  - 模块知识树节点一行代表一个模块或功能点。
  - 影响面候选项一行代表一个可追加到需求影响范围的资源。

## 验收标准

- AC-UI-001：管理员可在项目接入中心查看项目、仓库、客户基线、索引状态和模块知识库。
- AC-UI-002：前端提供 MCP 索引推送指引，明确平台不保存本机绝对路径。
- AC-UI-003：需求表单选择项目、客户线和模块后，可加载并展示推荐影响页面、接口、数据表、权限和文档。
- AC-UI-004：用户可将推荐影响面追加到需求影响范围字段，并保留人工编辑能力。
- AC-UI-005：需求详情中的“Agent 交接资料”可进入现有 artifact 版本页面，主流程不再强调“需求执行包”作为需求人员菜单概念。
- AC-UI-006：前端 harness 文档同步更新页面模块、接口调用契约和菜单语义。

## 约束与假设

- 第一版索引通过本地 Codex 或索引 Agent 调用 MCP tool 推送，前端只负责说明和展示结果。
- 平台后端会拒绝包含个人绝对路径的索引数据，前端也要在提示文案中明确这一边界。
- 如果后端索引接口字段调整，前端实现前必须先同步 companion 后端契约。
- 现有 RuoYi Vue 2 和 Element UI 技术栈不升级、不新增重量依赖。
