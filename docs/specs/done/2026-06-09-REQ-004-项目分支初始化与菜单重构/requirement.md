# 项目分支初始化与菜单重构前端需求说明

## 目标

将需求管理菜单从独立基础资料维护，调整为以项目初始化为主入口的工作流。仓库、分支和模块知识不再以“仓库管理、客户定制线、模块功能点”三个菜单暴露给用户，而是在项目维护弹窗、项目接入中心和 MCP 索引结果中沉淀。

## 范围

- 移除或隐藏左侧菜单中的仓库管理、客户定制线和模块功能点入口。
- 项目维护弹窗的仓库维护改为一个 Git 仓库地址一行，仓库名称可从 Git 地址自动推导，仓库类型由用户选择。
- 项目维护弹窗允许纯后端服务，只要求至少一条有效仓库，不再强制同时存在前端仓库和后端仓库。
- 分支配置作为项目初始化记录的核心维度展示，并回显后端生成的 MCP 识别 key。
- 项目列表只展示初始化摘要，不把按分支的初始化详情塞进大表格。
- 前端契约和 harness 文档同步把“客户定制线”语义调整为“项目分支”。

## 不做范围

- 不实现本地仓库扫描器。
- 不在前端保存本机绝对路径。
- 不物理删除旧页面文件，除非它们已无任何路由或构建引用。
- 不破坏需求列表、执行包、影响面推荐和 MCP 索引现有主链路。

## 验收标准

- AC-UI-001：左侧菜单不再显示仓库管理、客户定制线和模块功能点。
- AC-UI-002：新增项目时可只维护一个后端仓库并保存。
- AC-UI-003：填写 Git 地址后，仓库名称可自动从地址推导，用户仍可手动改名并选择仓库类型。
- AC-UI-004：分支配置展示 MCP key，提示 MCP 使用 key + Git 地址识别项目、分支和仓库。
- AC-UI-005：项目列表初始化状态对纯后端项目不再提示缺前端仓库。
- AC-UI-006：前端 harness 文档同步菜单、页面和契约变化。
