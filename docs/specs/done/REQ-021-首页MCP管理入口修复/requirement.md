# 首页MCP管理入口修复需求说明

## 背景

开发人员具备 `req:mcp:key:list` 权限，首页会展示 MCP 管理快捷入口。但点击后进入 404，影响开发人员创建和查看 MCP Key。

## 目标

- 修复首页 MCP 管理快捷入口路径，让开发人员能从首页进入 MCP 管理页。
- 增加轻量静态回归检查，防止后续再次把组件路径误写成路由 path。
- 同步模块文档，明确首页快捷入口使用后端菜单路由 path。

## 可行性评估

- 评估结论：可继续设计
- 主要风险：无
- 需需求人补充或调整：无
- 是否允许继续生成需求设计：是，问题定位清晰且影响范围小。

## 范围

本次包含：

- 修改首页快捷入口中 MCP 管理的跳转路径。
- 新增首页快捷入口静态检查脚本。
- 更新前端 harness 模块文档和 UI 契约。

本次不包含：

- 不调整后端菜单 SQL。
- 不调整 MCP 管理页面接口和权限。
- 不改动其他首页快捷入口。

## 影响范围

- 接口/API：否。
- 数据库/SQL：否。
- 权限/菜单：是，仅修正首页快捷入口跳转路径，不改变权限点。
- 页面/交互：是，开发人员首页点击 MCP 管理不再跳转到错误路径。
- 导出/异步/任务：否。

## 契约与数据口径

- 菜单路由：后端菜单脚本中 MCP 管理路由 path 为 `mcp-key`，在父路由 `requirement` 下完整路径为 `/requirement/mcp-key`。
- 组件路径：MCP 管理组件仍为 `requirement/mcpKey/index`，不得作为首页快捷入口 URL。
- 权限过滤：仍使用 `req:mcp:key:list` 控制是否展示 MCP 管理快捷入口。

## 验收标准

- AC-001：首页 MCP 管理快捷入口路径为 `/requirement/mcp-key`。
- AC-002：首页源码不再包含旧的 `/requirement/mcpKey` 快捷入口路径。
- AC-003：新增静态检查能防止旧路径回归。
- AC-004：前端生产构建通过。
- AC-005：harness 文档记录该入口路径规则。

- 现有动态菜单仍由后端返回，前端首页快捷入口必须使用菜单路由 path 而非组件路径。
