# 未办结任务清理与接入中心删除需求说明

## 背景

项目管理中“项目接入中心”和“项目维护”的信息仍大量重叠，维护页内分支知识库已经能承接接入中心展示的知识库能力。继续保留接入中心会造成入口和认知重复。仓库过程文档中也有已合入主干但未归档的任务，需要统一清理。

## 目标

- 项目列表只保留项目维护入口，不再展示“接入状态”或“项目接入中心”操作。
- 删除 `/requirement/project/detail` 隐藏路由和 `detail.vue` 页面文件。
- 保留项目维护页和分支知识库页签，项目初始化状态仍可在列表和维护页中查看。
- 更新前端模块知识库，删除接入中心作为长期入口的描述。
- 配合后端归档已合入主干但未办结的 active spec。

## 范围

本次包含：

- 修改项目列表操作列，移除接入中心跳转。
- 删除项目接入中心路由配置和页面文件。
- 改造静态检查，断言接入中心链路不存在。
- 更新前端模块知识库、执行报告和 Review 报告。
- 归档前端 `REQ-014-harness命名规则移除日期`。

本次不包含：

- 修改项目维护页的仓库、分支、初始化指令或知识库能力。
- 删除分支知识库页签。
- 修改后端接口、返回字段或权限标识。
- 直接调整本地平台库中的需求流转状态。

## 影响范围

- 接口/API：否，前端仍使用现有项目初始化和索引接口。
- 持久化结构/脚本：否。
- 权限/菜单：是，删除一个隐藏页签路由；权限标识不变。
- 页面/交互：是，项目列表不再提供接入中心入口。
- 导出/异步/任务：否。

## 契约与数据口径

- 接口路径和方法：无新增或删除。
- 请求参数：无变化。
- 响应字段：无变化。
- 数据粒度：项目管理列表一行代表一个项目；项目维护页是一项项目配置的主维护入口；分支知识库页签按项目分支展示知识库。

## 验收标准

- AC-UI-001：项目列表操作列不再包含“接入状态”或“项目接入中心”按钮。
- AC-UI-002：前端路由不再声明 `/requirement/project/detail` 和 `RequirementProjectDetail`。
- AC-UI-003：`src/views/requirement/project/detail.vue` 被删除，静态检查确认页面链路不存在。
- AC-UI-004：项目维护页和分支知识库页签仍保留路由与入口，不影响既有维护能力。
- AC-UI-005：前端模块知识库不再把“项目接入中心”记录为独立长期入口。
- AC-UI-006：前端文档检查、harness 完成态检查、静态检查和生产构建通过。
- AC-BE-001：后端 companion 完成已合并 active spec 归档并通过对应检查。

## 约束与假设

- 项目初始化上下文接口仍被项目维护和分支知识库使用，不能因为删除接入中心页面而删除接口封装。
- 没有明确需求记录 ID 时，不直接修改平台库中的需求状态。
- 本次仅删除重复入口，保留项目维护中的分支知识库相关能力。
