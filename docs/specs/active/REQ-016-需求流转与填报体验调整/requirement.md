# 需求流转与填报体验调整需求说明

## 背景

需求平台前端已经完成基础项目、需求和 Agent 资料页面，但当前体验仍带有通用后台模板痕迹：布局设置入口可切换出非目标布局，顶部多标签下隐藏页签返回依赖浏览器历史，需求表单还展示系统字段，需求状态流转也没有贴合“提需、编排、开发、验收”协作过程。目标用户是需求人员、开发人员和管理员。

## 目标

- 固定为白色左侧菜单布局，移除用户可见布局设置入口，并重新设计系统 logo。
- 修复隐藏页签在多标签场景下返回父菜单不稳定的问题。
- 简化新增和修改需求填报体验：不展示创建人 ID，修改时编号只读文本展示。
- 增加需求来源必填、业务背景纯文本填写和需求附件上传，粘贴图片或文件时作为附件上传，附件单文件最大 2MB。
- 收敛需求列表操作列，把 Agent 资料入口放到详情页内。
- 调整状态文案、按钮和详情页信息，让提需、生成需求设计、执行任务、验收闭环清晰可操作。
- 增加返修流程和资料历史版本展示，让返修版本重新进入验收流程。
- 适配三类角色权限体验：需求人员只进入首页、需求列表和使用统计；开发人员可进入首页、需求列表、MCP 管理和使用统计；管理员保留全部功能。
- 调整详情页 Agent 交接资料包布局，直接嵌入当前需求详情，避免底部需求设计、执行计划内容与资料包重复或重叠。

## 范围

本次包含：

- `src/layout/**`、`src/settings.js` 和侧边栏 logo/布局设置相关前端调整。
- `src/plugins/tab.js`、需求详情和维护页签返回逻辑调整。
- `src/views/requirement/demand/*.vue` 和 `src/api/requirement/demand.js` 的需求状态、按钮、表单和详情调整。
- `src/components/FileUpload/index.vue` 的 2MB 上传边界、文件类型校验与需求专用上传接口适配。
- 前端 harness 模块和契约文档同步。
- 需求详情内嵌资料包标题、文档分组、历史版本和指令按钮文案调整。

本次不包含：

- 后端角色 SQL、需求来源/附件字段和上传大小硬限制由 companion 后端实现。
- 重做 Agent 交接资料编辑页的整体信息架构。

## 影响范围

- 接口/API：是，新增或复用需求状态流转接口；前端不再提交创建人 ID 和需求编号。
- 数据库/SQL：前端不执行 SQL；需求来源和附件字段由后端 companion 迁移脚本维护。
- 权限/菜单：是，移除可见布局设置入口；前端按后端菜单权限自然收敛角色可见菜单。
- 页面/交互：是，布局、Logo、隐藏页签返回、需求列表、需求维护、需求详情。
- 导出/异步/任务：否。

## 契约与数据口径

- 需求新增：前端提交需求业务字段，不提交 `creatorId` 和 `demandNo`；后端自动生成编号和创建人。
- 需求来源：新增和修改需求均必填，前端保存 `demandSource` 文本值并在列表、详情显示原文。
- 业务背景与附件：业务背景为普通多行文本，不承载富文本或图片；在业务背景框内粘贴图片或文件时自动上传为附件。附件上传使用 `/requirement/demand/upload`，前端上传组件单文件限制 2MB，附件最多 5 个，仅支持 Word、Excel、PDF、TXT、PPT 和常见图片格式。
- 需求编号展示：新增时不展示；修改和详情只作为文本展示，不以 input 呈现，不允许用户编辑。
- 状态流转：列表和详情展示新状态文案；需求人员可从未提交提交需求、确认需求设计、提交返修或确认验收，开发人员可提交需求设计、开始开发、提交验收或提交返修验收；待验收可进入返修，返修完成后重新提交验收。
- 返回逻辑：隐藏页签关闭或返回时优先跳转路由 `meta.activeMenu` 对应父菜单。
- 资料版本：详情页读取执行包版本列表，展示需求草稿、需求设计、执行计划、执行报告和 Review 报告等历史版本；返修资料通过新增版本保留历史。
- 指令生成：详情页不展示协作工具栏，不展示复制内容；开发人员和管理员在待生成需求设计阶段展示一个“生成需求评估与设计”按钮，后端指令语义为在需求设计阶段创建任务分支、先生成并回写需求可行性评估，评估允许后再生成详细需求设计；在待执行开发、开发中、返修中和待验收阶段展示一个“生成执行任务指令”按钮，后者包含执行计划、执行报告和 Review 报告三个一次性 actionToken。

## 验收标准

- AC-001：系统不再展示布局设置入口，默认白色左侧菜单布局，侧边栏展示新的 ReqFlow logo。
- AC-002：多个 tab 同时打开时，需求详情、需求维护等隐藏页签点击返回或关闭后跳转到 `/requirement/demand` 父菜单。
- AC-003：新增需求页面不展示创建人 ID 和需求编号；提交请求不携带用户填写的 `creatorId` 和 `demandNo`。
- AC-004：修改需求页面把需求编号作为文本展示，不能编辑，不使用 input 样式。
- AC-005：需求列表操作列不展示 Agent 资料入口；详情页内嵌展示需求设计、执行计划等资料摘要。
- AC-006：需求状态文案和操作按钮符合新流转：未提交、待生成需求设计、需求设计待确认、待执行开发、开发中、待验收、已完成，并按需求人员、开发人员和管理员角色过滤展示。
- AC-007：前端 harness 模块与契约文档同步记录布局、返回、表单和状态流转约束。
- AC-008：侧边栏 logo 图形和文字垂直对齐；列表和详情内流程流转按钮采用统一的状态按钮风格。
- AC-009：需求详情内流程确认按钮与复制、跳转类工具按钮有明显视觉区分，流程确认按钮集中放在详情头部状态区。
- AC-010：待验收状态支持提交返修，返修状态支持提交返修验收，返修版本按同一流程重新进入验收。
- AC-011：需求详情内嵌当前需求的 Agent 交接资料包，展示需求可行性评估、需求设计、执行计划和执行报告等各项文档及历史版本，并在开发、返修、验收阶段提供执行任务指令复制入口。
- AC-012：初始化、生成需求设计和执行任务指令中的 actionToken 由后端生成，前端只复制后端返回内容；页面提示 24 小时内有效、仅可使用一次，且不持久化明文 actionToken。
- AC-013：从需求详情进入 Agent 交接资料时，不展示查询、手动生成、加载最新或保存新版本按钮，只展示当前需求标题和各项文档内容。
- AC-014：需求详情底部不再重复展示独立的“需求设计与执行计划”预览块，资料内容统一收敛到内嵌 Agent 交接资料包。
- AC-015：需求详情状态文案、确认按钮和复制按钮区分“生成需求设计”和“执行任务”，不再使用“需求说明和执行计划”混合文案。
- AC-016：内嵌 Agent 交接资料包使用当前需求标题作为区块标题，按需求设计、执行计划、执行报告等文档分组展示，并在移动端和桌面端不与底部返回按钮或业务内容重叠。
- AC-017：前端模块和契约文档记录三类角色的菜单可见性和流程按钮可见性预期，需求人员没有 MCP 管理和 Agent 交接资料独立菜单入口，仍可在需求详情查看当前需求资料。
- AC-018：新增和修改需求时需求来源为必填文本输入；业务背景使用普通文本框，粘贴图片或文件会作为附件上传；需求附件上传单文件不超过 2MB，详情页可查看来源、纯文本背景和附件。
- AC-019：首页快捷入口按权限过滤，需求人员没有 MCP 管理权限时不得显示 MCP 管理快捷入口。
- AC-020：管理员拥有删除需求入口；需求人员和开发人员不显示删除按钮，流程按钮同时按角色和按钮权限过滤。
