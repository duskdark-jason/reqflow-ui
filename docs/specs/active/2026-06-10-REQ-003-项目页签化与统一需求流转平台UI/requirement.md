# 项目页签化与统一需求流转平台UI 需求说明

## 背景

当前项目管理已经把项目、仓库、项目分支和初始化状态集中到维护弹窗，但弹窗承载的信息越来越重，分支知识库还依赖表格展开展示，项目分支识别字段仍以裸 `MCP Key` 形式露出。系统品牌和首页仍残留若依默认信息，整体视觉也更像通用后台模板，不符合需求流转平台的业务定位。

本需求面向管理员、需求人员和开发人员，把项目维护、分支初始化指令、知识库详情、登录页和后台首页统一调整为“统一需求流转平台”的浅色工作台体验。

## 目标

- 项目新增和维护从弹窗改为可保留在 TagsView 中的新页签。
- 分支识别信息从裸 `MCP Key` 改为可复制的初始化指令，指令包含简短提示词和加密唯一 token。
- 分支知识库详情从表格展开改为新页签详情页。
- 系统整体采用浅色需求管理风格，登录页有匹配背景图且登录框右移。
- 清理用户可见若依信息，把系统命名为统一需求流转平台，首页改造成需求流转看板。

## 范围

本次包含：

- 新增或改造项目维护页签，承载原 `ProjectInitWizard` 的项目信息、代码仓库、项目分支和初始化摘要能力。
- 项目管理列表的新增、维护按钮改为打开项目维护页签；保存后支持返回列表或进入项目接入中心。
- 分支配置中展示“初始化指令”复制入口，不再把 `MCP Key` 作为主视觉字段。
- 新增项目分支知识库详情页签，按项目和分支展示模块、索引批次、页面、接口、数据表、权限、文档等知识内容。
- 调整登录页、后台首页、系统标题、页脚、导航右侧入口、前端包描述和用户可见品牌文案。
- 更新前端 harness 契约、模块文档和领域说明。

本次不包含：

- 不重命名 Java 包名、前端工具函数文件名或历史兼容组件内部名称。
- 不新增前端依赖，不升级 Vue、Element UI、ECharts 或构建工具。
- 不实现浏览器本地仓库扫描，不保存个人本机绝对路径。
- 不把人员 MCP 访问 Key 与项目分支初始化 token 混用。
- 不在未授权状态创建任务分支、实现代码、提交或进入 Review。

## 影响范围

- 接口/API：是，前端需要消费后端新增或扩展的分支指令字段；首页继续复用 `/requirement/statistics/**`。
- 数据库/SQL：前端无直接数据库变更；依赖后端 companion 判断 token 是否需要落库。
- 权限/菜单：是，新增隐藏页签路由继续使用 `req:project:query`、`req:project:add`、`req:project:edit` 和 `req:index:list`。
- 页面/交互：是，项目管理、项目维护页、项目接入中心、知识库详情、登录页、后台首页、导航栏和页脚会调整。
- 导出/异步/任务：否。

## 契约与数据口径

- 项目维护页签路由建议为 `/requirement/project/maintain`，新增时不带 `projectId`，编辑时通过 query 携带 `projectId`。
- 分支知识库详情页签路由建议为 `/requirement/project/knowledge`，通过 query 携带 `projectId`、`variantId` 和可选 `repoId`。
- 项目维护保存仍调用 `/requirement/project/init` 的 `POST` 和 `PUT`。
- 初始化指令来自后端 `variants` 行级返回字段，建议前端消费 `initInstruction` 或 `instructions.init`；字段应包含 `actionType`、`token`、`prompt`、`content`、`expireTime` 和 `copyLabel`。
- 分支知识库详情优先复用 `/requirement/index/batch/list`、`/requirement/index/module/tree` 和后续 companion 扩展的知识详情接口。
- 首页看板复用 `src/api/requirement/statistics.js` 中的概览、项目排行和用户使用统计接口。
- 数据粒度：项目列表一行一个项目；项目维护页一次编辑一个项目初始化上下文；分支指令一条对应一个项目分支和一个 MCP 动作；知识库详情以 `projectId + variantId` 为过滤粒度。

## 验收标准

- AC-UI-001：点击项目管理“新增”或“维护”会打开新的项目维护页签，不再出现项目维护 dialog。
- AC-UI-002：项目维护页签完整支持项目基础信息、代码仓库、项目分支、初始化摘要、保存、保存并进入接入中心。
- AC-UI-003：分支配置主展示字段为“初始化指令”，提供复制按钮，复制内容包含简短触发提示词和加密唯一 token，不再要求用户复制裸 `MCP Key`。
- AC-UI-004：项目分支知识库详情通过新页签展示，不再依赖项目分支表格 expand 展开详情。
- AC-UI-005：整体 UI 为浅色需求管理风格，登录页使用匹配背景图，登录框在桌面端右移且移动端可用。
- AC-UI-006：用户可见若依信息已清理，系统名称展示为统一需求流转平台。
- AC-UI-007：首页展示统一需求流转平台看板，包含需求、执行资料、项目排行、活跃用户或待处理状态，不再展示若依默认说明和更新日志。
- AC-UI-008：前端 harness 文档同步记录项目页签化、初始化指令、知识库详情页、品牌 UI 和验证路径。

## 约束与假设

- 当前需求属于平台自身建设模式，尚未通过需求平台 Key 驱动。
- 当前前端仓库处于 `main` 分支；计划阶段不得直接实现代码。
- 后端 companion 需要提供或确认初始化指令 token 生成、校验和响应字段。
- UI 设计采用已有 `frontend-design` 和 `ui-ux-pro-max` 技能约束：浅色、多信息密度、明确层级、低装饰、高可读性。
- 若依清理以用户可见品牌为准，底层框架包名和历史工具注释不在本次重命名范围内。
