# 项目管理初始化向导前端需求说明

## 背景

REQ-002 已把项目接入中心、仓库和客户基线维护入口、MCP 索引指引、影响面推荐基础能力落到前端，但仍没有达到最初设想的项目管理菜单体验。管理员在“新增项目”时仍只能维护项目基础字段；前后端仓库、统一客户基线、模块功能点和索引状态仍需要进入项目接入中心或其他菜单补齐。

本需求要把后台“项目管理”改造成真正的项目初始化入口：管理员点击新增或编辑项目后，在一个维护弹窗内填写项目名称、前端仓库、后端仓库和后续可持续维护的分支列表，并看到模块功能点初始化状态。平台不保存本机目录；本机目录仅作为本地索引器执行时的临时输入或提示，不进入后端保存。

## 目标

- 将项目管理新增/编辑改造成单页项目维护弹窗，而不是分步向导或单一项目基础字段弹窗。
- 在项目管理菜单内同时维护项目基础信息、前端仓库、后端仓库、分支中文标签和真实分支名。
- 在项目列表中展示初始化状态，让管理员能快速识别项目是否具备仓库、分支、模块知识和索引批次。
- 在维护弹窗内展示 MCP 索引下一步提示，但不把 MCP 作为管理员必须理解的唯一入口。
- 保留项目接入中心详情页作为项目后续维护入口，不重复造一个孤立页面。

## 范围

本次包含：

- 改造 `src/views/requirement/project/index.vue` 的新增/编辑入口，使用一个维护弹窗承载项目初始化信息。
- 新增或调整初始化 API 封装，调用后端 companion 的项目初始化聚合接口。
- 维护弹窗包含：项目信息、代码仓库、分支列表、模块初始化状态和保存操作。
- 仓库步骤默认给出前端仓库和后端仓库两行，可增删其他仓库；只保存团队共享 Git 远端和默认分支。
- 分支列表默认给出一条主线分支，可维护对需求人员展示的中文标签和真实分支名，后续可持续新增分支行。
- 项目列表展示初始化状态标签或进度摘要，并提供“继续初始化/接入中心”入口。
- 更新前端 harness 文档，说明项目管理菜单是初始化主入口。

本次不包含：

- 浏览器直接扫描本机仓库或读取本机文件系统。
- 前端保存个人绝对路径。
- 重新实现 MCP 索引器。
- 新增复杂可视化依赖或替换 Element UI。
- 移除已有仓库管理、客户定制线和模块功能点菜单。

## 影响范围

- 接口/API：是，新增项目初始化聚合 API 封装。
- 数据库/SQL：否，前端不直接改数据库；依赖后端 companion。
- 权限/菜单：是，项目管理新增/编辑维护弹窗需要组合项目、仓库、分支维护权限。
- 页面/交互：是，项目列表、新增项目、编辑项目、项目接入中心入口都会调整。
- 导出/异步/任务：否。

## 契约与数据口径

- API 文件建议：
  - `src/api/requirement/projectInit.js` 封装 `getProjectInit`、`addProjectInit`、`updateProjectInit`。
- 页面入口：
  - `src/views/requirement/project/index.vue`：项目管理列表、初始化状态、维护弹窗入口。
  - `src/views/requirement/project/detail.vue`：项目接入中心详情页，保留作为后续维护和索引查看入口。
- 维护弹窗：
  - “项目信息”：项目名称、项目编码、负责人、说明、状态。
  - “代码仓库”：仓库名称、仓库类型、Git 地址、默认分支、状态；默认包含前端和后端仓库。
  - “分支列表”：中文标签、真实分支名、状态和备注；中文标签面向需求人员展示，真实分支名用于索引、影响面推荐和后续开发基准。
  - “模块初始化状态”：展示模块数、索引批次、最近索引和下一步 MCP 指引；不保存本机目录。
- 数据粒度：
  - 列表一行一个项目，并展示该项目初始化状态。
  - 维护弹窗一次编辑一个项目的初始化上下文。
  - 仓库一行一个团队共享 Git 仓库。
  - 分支一行代表一个可供需求人员选择的项目分支配置。

## 验收标准

- AC-UI-001：管理员点击项目管理“新增项目”后看到单页项目维护弹窗，而不是分步初始化向导或只有项目基础字段的弹窗。
- AC-UI-002：维护弹窗可维护项目名称、项目编码、前端仓库、后端仓库、分支中文标签和真实分支名。
- AC-UI-003：向导和项目列表明确提示平台不保存本机目录，只保存共享 Git 信息和基线分支。
- AC-UI-004：项目列表展示初始化状态，能区分缺仓库、缺分支、缺模块知识、已完成基础初始化。
- AC-UI-005：编辑已有项目时可回显项目、仓库和分支列表，并保存修改后仍返回项目列表或进入接入中心。
- AC-UI-006：前端 harness 文档同步更新项目管理菜单、维护弹窗和接口调用契约。
