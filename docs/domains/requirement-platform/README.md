# 需求管理平台前端领域入口

## 业务边界

本领域承接需求管理平台前端能力，包括项目初始化、项目分支、MCP 索引知识、需求提交、执行包查看和使用统计页面。

## 当前状态

- 需求管理业务页面已收敛为项目管理、分支知识库、需求、Agent 交接资料和统计页面；仓库管理、客户定制线和模块功能点不再作为左侧菜单入口。
- 项目管理已改造为项目初始化主入口，新增和修改项目时通过项目维护页签维护项目基础信息、代码仓库、项目分支、初始化指令和模块初始化状态。
- 项目维护页签负责项目配置、初始化指令复制和初始化状态查看；分支知识库页签负责查看索引批次和模块知识库。
- 分支知识库详情通过独立页签展示，按 `projectId + variantId` 过滤索引批次和模块知识，不再依赖表格展开行。
- 新增和编辑需求时，项目分支只能从已初始化完成的分支中选择；未完成模块知识和仓库索引初始化的分支不进入提交表单下拉。
- 前端 API 封装已覆盖 `/requirement/project/init/**`、基础 CRUD、仓库索引、需求、执行包和统计接口。
- 用户可见品牌为“统一需求流转平台”，首页是需求流转看板，登录页使用浅色需求管理风格背景图。
- 前端不保存个人本机仓库目录；本机目录只属于本地索引器临时读取范围。

## 后续维护

- 新增页面、路由、按钮权限或 API 调用时，同步补充 `docs/ai-harness/modules/` 或 `docs/ai-harness/contracts/`。
- 新增长期有效交互规则时，同步补充 `docs/ai-harness/decisions/`。
